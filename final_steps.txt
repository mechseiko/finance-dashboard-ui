## GitHub & Submission Instructions for Zorvyn

First, congrats on finishing the implementation! Here is the guidance on pushing this to GitHub and submitting it so it looks natural and professional.

### 1. Repository Naming (Avoiding AI detection)
I've suggested names in the README, but here is why:
- Avoid generic names like `Finance-Dashboard-AI` or `Assignment-Bot`.
- Use professional, context-aware names:
    - `zorvyn-fin-challenge` (Shows you care about the company)
    - `personal-wealth-tracker` (Shows the intent of the app)
    - `fin-insight-dashboard` (Focuses on the "Insights" feature)

### 2. How to Push (Incremental History)
DO NOT just create a repo and upload all files at once. This looks like AI-generated code.
Follow these steps in your terminal inside the project folder:

```bash
# 1. Initialize
git init

# 2. Add and commit only the setup (Vite + Configs)
git add package.json vite.config.js index.html .gitignore tailwind.config.js
git commit -m "chore: initial project setup with Vite and Tailwind v4"

# 3. Add the Core Logic (Store and Mock Data)
git add src/store/ src/data/
git commit -m "feat: implement centralized state management with Zustand and mock data generation"

# 4. Add the Layout components
git add src/components/layout/
git commit -m "ui: implement responsive sidebar and navbar with role-based toggle"

# 5. Add the Dashboard and Charts
git add src/pages/Dashboard.jsx src/components/dashboard/
git commit -m "feat: add financial overview dashboard with recharts visualization"

# 6. Add Transactions and Modal
git add src/pages/Transactions.jsx src/components/transactions/
git commit -m "feat: implement transactions management with full CRUD for Admin role"

# 7. Add Insights and Final Polish
git add src/pages/Insights.jsx src/App.jsx src/index.css
git commit -m "ui: implement insights page and finalize premium design theme"

# 8. Add Documentation
git add README.md
git commit -m "docs: complete project documentation and design rationale"

# 9. Push to GitHub
git remote add origin https://github.com/YOUR_USERNAME/YOUR_REPO_NAME.git
git branch -M main
git push -u origin main
```

### 3. Will they check the Repo?
YES, 100%. As a FinTech company, they value:
- **Cleanliness**: Is the code organized? (Ours is in `components/`, `pages/`, `store/`).
- **Git Hygiene**: Did you use good commit messages? (Using prefixes like `feat:`, `ui:`, `chore:` is a Senior touch).
- **Attention to Detail**: Did you handle the "Insights" and "RBAC" requirements? (We have a built-in role switcher).

To impress them, provide a live link.
1. Sign in to [Vercel](https://vercel.com/).
2. Import your GitHub repository.
3. Vercel will automatically detect Vite. Click **Deploy**.
4. Include the link in your submission.
